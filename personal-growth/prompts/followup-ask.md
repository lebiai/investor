# 收尾追问模板

> 每个 skill 工作流完成后，追加 1 行追问。不独立步骤、不打断、可忽略。

## sector-analysis

"分析完了。这次有没有什么让你意外的发现？"

回答解析规则：
- 含"发现""不一样""没想到""反转"等 → 类型标记为 `pattern`
- 含"错了""没看到""忽略了"等 → 类型标记为 `lesson`
- 空回答或无内容 → 跳过，不记录

## deal-review

"审阅完了。这个BP里有没有之前没见过的新陷阱？"

回答解析规则：
- 含"陷阱""风险""问题""不对劲"等 → 类型标记为 `pattern`，标签含 deal-review
- 如描述了一种新模式 → 同时通知 deal-review 追加到 `data/deals/patterns.md`
- 空回答 → 跳过

## deal-sourcing

"扫描完了。有没有让你眼前一亮的新标的？"

回答解析规则：
- 含"不错""有意思""可以看看"等积极信号 → 类型标记为 `insight`，标签含领域名
- 空回答 → 跳过

## content-prod

"写完了。发现知识库里缺了什么信息？"

回答解析规则：
- 有具体缺失描述 → 类型标记为 `lesson`，标签含缺失领域
- "不缺"或空 → 跳过

## research-digest

"整理完了。有没有新旧信息矛盾的地方？"

回答解析规则：
- 有矛盾描述 → 类型标记为 `cross-link`，标签含两个领域名
- 无矛盾 → 跳过

## watch

"对比完了。最大的变化是什么？"

回答解析规则：
- 有变化描述 → 类型标记为 `insight`，标签含条目名
- "没变化"或空 → 跳过

## 通用回答解析

用户不是在 skill 收尾时回答，而是主动说"记录一下""我注意到"：

1. 识别意图 → 追问："属于模式/教训/原则还是跨领域连接？"
2. 用户给出分类后 → 追问："要给什么标签？"
3.存入 `inbox/timeline.md`，状态 `pending`

# 周报追问框架

## 统计阶段

列出本周记录：

```
📊 本周概览
- 工作活动：N 次（赛道分析/审阅/扫描/写作/整理）
- 碎片记录：N 条（待蒸馏）
- 关注变化：N 条（watch 动态）
```

## 询问用户

"你这周有 N 条碎片 + N 次工作活动，要花 5 分钟整理吗？"

用户说"好" → 执行蒸馏流程。
用户说"下次"或忽略 → 下周继续累积。

## 蒸馏流程

### Step 1: inbox 处理

从 `inbox/timeline.md` 按时间顺序逐条处理：

```
当前：第 1/N 条
"{原文}"
标签：{建议标签}
要我补充什么吗？（输入或回车跳过）
```

完成后：`inbox/timeline.md` 该行状态改为 `distilled`。

### Step 2: 写入 vault

将每条蒸馏后的记录转到 `vault/timeline.md`：

```markdown
| {日期} | {类型} | {标签} | {用户确认的摘要} | {来源} | vault/entries/{id}.md |
```

正文（如有详细内容）写入 `vault/entries/{id}.md`：
```markdown
---
type: {类型}
created: {日期}
tags: [{标签}]
source: {来源}
---
详细内容...
```

### Step 3: 周报

```markdown
# 周回顾 — {年份}-W{周数}

## 1. 这周有没有 moment 让你觉得"之前想错了"？

{用户回答}

## 2. 这周注意到的 1 个重复模式

{用户回答}

## 3. 这周做的决定里，哪个信息量最大？

{用户回答}

## 4. 下周想验证的 1 个假设

{用户回答}
```

存入 `pulse/weekly/{年份}/W{周数}.md`。

### Step 4: 更新索引

```markdown
| {年份} | W{周数} | pulse/weekly/{年份}/W{周数}.md |
```

追加到 `pulse/index.md`。

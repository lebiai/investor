# 模板生成逻辑

## 触发场景

用户说：
- "用[模板名]生成一份"
- "套[模板名]模板"
- "按会议纪要模板帮我写一份"

## 生成流程

### Step 1: 查找模板

1. 查 `knowledge-base/templates/index.md` → 匹配模板名
2. 未找到 → "模板库中没有[模板名]，已有模板：[列表]"
3. 找到 → 读取 `knowledge-base/templates/parsed/模板名.md`

### Step 2: 收集填写内容

根据模板解析文件中的"填写字段"逐项询问用户：

```
模板 [模板名] 需要以下信息：
1. 会议主题 → [用户输入]
2. 会议日期 → [用户输入]
3. 参会人员 → [用户输入]
...
一次告诉我，或我逐个问
```

如果用户一次性说全，直接解析使用；否则逐个追问。

### Step 3: 生成内容

按模板结构填充：

1. 按照解析文件中的"正文结构"组织内容
2. 将用户提供的信息填入对应字段
3. 表格保持原模板的列数和结构
4. 列表保持原模板的层级

### Step 4: 交付

```
# [模板名] — [标题]

[按模板结构输出的完整内容]
```

### Step 5: 保存（可选）

询问用户：
- 是否保存到 outputs/ 目录
- 是否保存到 knowledge-base/ 关联项目

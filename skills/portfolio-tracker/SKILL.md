# 投后管理

## 触发词

- "我的投资组合" / "投后看板" — 全景组合概览
- "[公司名]最近怎么样" — 单家公司深度状态
- "把XX公司加入投后管理" — 新增组合公司
- "XX公司的里程碑进度" / "XX公司里程碑" — 里程碑追踪
- "最近有什么预警" / "投后预警" — 风险警报汇总
- "帮我看下周XX公司的董事会" — 董事会议程准备
- "XX公司上市进展" / "XX公司退出进展" — 退出路径跟踪
- "XX公司上月营收达标了" / "XX公司完成B轮" — 状态更新
- "XX公司管理层变动了" — 重大事件录入

## 数据来源

本 skill 的数据来源：

| 来源 | 调用方式 | 说明 |
|------|---------|------|
| ① 用户输入 | 直接对话接收 | 组合公司状态更新、里程碑完成、重大事件 |
| ② [skill:agent-reach] 搜索 | 使用 agent-reach 获取互联网数据（`[skill:agent-reach]`） | 新闻/融资/上市/竞品动态 |
| ③ `data/` 知识库 | 直接读取本地 data/ 目录 | companies/ sectors/ 历史档案 |
| ④ `data/portfolio/` 自身库 | 读取历史投后记录 | 越用越精准 |

## 前置条件

部分功能（看板/预警/里程碑追踪）依赖已录入的组合公司数据，首次使用为空状态。
说"把XX公司加入投后管理"开始录入。

## 工作流

### Phase 1: 新增组合公司 [来源: ①用户输入]

用户说"把XX公司加入投后管理"：

1. **初步建档：**
   - 询问基本信息：
     - 投资轮次 / 投资金额 / 股权占比 / 投资日期
     - 赛道 / 细分领域
     - 核心联系人（CEO/CFO）
   - 加载 `references/portfolio-framework.md` 建档模板
   
2. **知识库补充 [来源: ③知识库 + ②agent-reach搜索]：**
   - rg 搜索 `data/companies/` 和 `data/sectors/` 中是否有历史档案
   - 如匹配 → 自动关联历史数据
   - 调用 [skill:agent-reach] 搜索公司最新动态
   
3. **初始里程碑设定：**
   - 加载 `references/milestone-engine.md`
   - 询问关键里程碑：下一轮融资预期 / 营收目标 / 产品上线 / 盈亏平衡时间
   - 引导用户设定 3-5 个核心里程碑
   
4. **写入知识库：**
   - 写入 `data/portfolio/[公司名].md`
   - 更新 `data/portfolio/index.md`
   - 更新 `data/companies/` 对应档案（标注已投）
   - 更新 `data/index.md`
   
5. **输出确认：**
   ```
   ✅ XX公司已加入投后管理
   
   基本信息：
   ├── 投资轮次：A轮 | 金额：¥XXXX万 | 占比：X%
   ├── 赛道：固态电池
   ├── 投资日期：2026-07
   └── 核心联系人：张三(CEO)
   
   初始里程碑（可调整）：
   ├── 🔴 里程碑1：[描述] — 预期：2026-Q3
   ├── 🟡 里程碑2：[描述] — 预期：2026-Q4
   └── 🟢 里程碑3：[描述] — 预期：2027-Q1
   ```

### Phase 2: 组合全景看板 [来源: ①用户输入 + ④自身库]

用户说"我的投资组合" / "投后看板"：

1. **读取组合索引：** 加载 `data/portfolio/index.md`
2. **遍历状态：** 对每个组合公司，读取其档案的最新状态
3. **自动预警扫描 [来源: ②agent-reach搜索]：**
   - 加载 `references/alert-rules.md`
   - 对每个公司调用 [skill:agent-reach] 搜索最新动态
   - 与档案中的里程碑 / 预期对比
   - 标记异常（管理层变动 / 融资延迟 / 营收未达标 / 竞争恶化）
4. **输出看板：**
   ```
   📊 投资组合全景（截至 YYYY-MM-DD）
   
   🟢 健康（N家）
   ├── XX公司 — 里程碑进度：3/5 | 下个里程碑：YYYY-MM
   └── XX公司 — 里程碑进度：2/3 | 下个里程碑：YYYY-MM
   
   🟡 需关注（N家）
   ├── XX公司 — ⚠️ 营收未达标（预期¥X，实际¥Y）
   └── XX公司 — ⚠️ 下一轮融资延迟2个月
   
   🔴 预警（N家）
   ├── XX公司 — 🚨 管理层变动：CTO离职
   └── XX公司 — 🚨 现金流预警：可维持X个月
   
   组合统计：
   ├── 总投资额：¥XXXX万 | 当前估值：¥XXXX万 | MOIC：X.Xx
   ├── 已退出：N家 | 退出总额：¥XXXX万
   ├── 正常运营：N家 | 需要关注：N家 | 预警：N家
   └── 最近更新：YYYY-MM-DD
   ```

### Phase 3: 单公司深度状态 [来源: ①用户输入 + ②agent-reach搜索 + ④自身库]

用户说"[公司名]最近怎么样"：

1. **读取档案：** `data/portfolio/[公司名].md`
2. **里程碑检查：** 加载 `references/milestone-engine.md`，比对实际 vs 预期
3. **外部扫描 [来源: ②agent-reach搜索]：** 搜索公司最新新闻/融资/竞争动态
4. **预警检查 [来源: ④自身库]：** 读取 `data/portfolio/alerts.md` 中关联该公司的预警
5. **输出深度状态：**
   ```
   📋 XX公司 — 投后状态
   
   基本信息：
   ├── 赛道：固态电池
   ├── 投资轮次：A轮 | ¥XXXX万 | 占比X%
   ├── 投资日期：2026-07 | 最近更新：2026-07-16
   └── 核心联系人：张三(CEO)
   
   里程碑进度：
   ├── 🟢 [里程碑1]已完成（YYYY-MM-DD）
   ├── 🟡 [里程碑2]进行中 — 预期：2026-Q3 | 当前状态：正常推进
   └── 🔴 [里程碑3]延迟 — 预期：2026-Q2 | 实际：Q3 | 原因：技术卡点
   
   最新动态（[来源: agent-reach]）：
   ├── 新闻：XX公司获得YY订单（来源）
   └── 竞品：ZZ公司完成B轮融资（可能影响竞争格局）
   
   预警状态：🟡 需关注
   └── 营收未达标：预期Q2 ¥X，实际¥Y（差异Z%）
   
   下步行动建议：
   ├── 🎯 跟进营收未达标原因
   └── 🎯 关注竞品融资后的市场变化
   ```

### Phase 4: 状态更新 / 重大事件录入 [来源: ①用户输入]

用户提供公司状态更新（如"XX公司上月营收达标了"/"XX公司管理层变动了"）：

1. **识别事件类型：** 里程碑完成 / 财务更新 / 管理变动 / 融资进展 / 产品上线 / 其他
2. **信息补充：** 询问关键细节（日期 / 数据 / 影响评估）
3. **档案更新：**
   - 更新 `data/portfolio/[公司名].md`（追加状态更新到 timeline 部分）
   - 如为里程碑完成 → 自动检测下个里程碑是否到期，引导设定新的
   - 如为预警事件 → 写入 `data/portfolio/alerts.md`
4. **组合影响评估：** 加载 `references/alert-rules.md`，判断该事件是否影响组合整体
5. **输出确认：** "已记录，XX公司状态已更新"

### Phase 5: 预警系统 [来源: ②agent-reach搜索 + ④自身库]

用户说"最近有什么预警" / 自动在看板输出时附带：

1. 加载 `data/portfolio/alerts.md` → 所有活跃预警
2. 加载 `references/alert-rules.md` → 与最新状态交叉比对
3. 调用 [skill:agent-reach] 搜索各组合公司最新新闻 → 识别潜在新预警
4. 按严重度排序输出：
   ```
   🚨 投后预警（YYYY-MM-DD）
   
   🔴 高危（需立即关注）
   ├── XX公司 — CTO离职 | 发现日期：YYYY-MM-DD | 影响：技术路线不确定性
   └── XX公司 — 现金流仅剩X个月 | 发现日期：YYYY-MM-DD
   
   🟡 关注
   ├── XX公司 — 竞品完成大额融资 | 发现日期：YYYY-MM-DD
   └── XX公司 — 核心客户流失 | 发现日期：YYYY-MM-DD
   
   ✅ 已关闭（N条）
   └── XX公司 — 营收未达标（已解决，YYYY-MM-DD）
   ```

### Phase 6: 董事会准备 [来源: ①用户输入 + ②agent-reach搜索 + ③知识库]

用户说"帮我看下周XX公司的董事会"：

1. 加载 `references/board-prep-template.md`
2. **数据整合 [来源: ③知识库 + ②agent-reach搜索]：**
   - 读取公司档案：里程碑进度 / 财务状态 / 历史纪要
   - 读取 `data/sectors/` → 赛道最新动态
   - 调用 [skill:agent-reach] 搜索竞品/行业最新信息
3. **生成董事会简报：**
   
   ```
   📋 XX公司 — 董事会简报 | YYYY-MM-DD
   
   一、公司整体状态
   ├── 当前估值：¥XXXX万 | 上次估值：¥XXXX万 | 变动
   ├── 里程碑进度：X/Y
   └── 预警状态：🟢/🟡/🔴
   
   二、经营数据（最近季度）
   ├── 营收：¥XXX万 | YoY：X%
   ├── 毛利：X% | 趋势：↑/↓/→
   ├── 亏损：¥XXX万 | 收窄/扩大
   └── 现金流：可维持X个月
   
   三、关键事项
   ├── 完成/未完成的里程碑及原因
   ├── 重大变化（管理层/股东/客户）
   └── 需要董事会决策的事项
   
   四、行业对标
   ├── 竞品动态
   ├── 赛道趋势
   └── 投资人关注点
   
   五、建议讨论议程
   1. ...
   2. ...
   ```

### Phase 7: 退出监控 [来源: ①用户输入 + ②agent-reach搜索]

用户说"XX公司上市进展" / "退出进展"：

1. 加载 `references/exit-monitor.md`
2. **数据整合 [来源: ③知识库 + ②agent-reach搜索]：**
   - 读取公司档案中的退出规划
   - 调用 [skill:agent-reach] 搜索 IPO/并购/SPAC 相关进展
3. **输出退出状态：**
   ```
   📋 XX公司 — 退出路径跟踪
   
   规划退出路径：IPO（目标：2027-Q2）
   ├── 当前阶段：Pre-IPO
   ├── 券商选择：已聘请YY券商
   ├── 审计进展：正在进行 2026 年度审计
   └── 预期估值：¥XXX-XXX万
   
   备选路径：战略并购
   └── 潜在买方：ZZ集团（已初步接触）
   
   最近动态（[来源: agent-reach]）：
   ├── 同赛道XX公司已提交招股书
   └── 监管层对行业IPO态度：正面
   ```

### Phase 8: 知识库入库

自动更新（用户无需感知）：

1. 写入 `data/portfolio/[公司名].md`（每次状态更新追加 timeline）
2. 更新 `data/portfolio/index.md`
3. 更新 `data/portfolio/alerts.md`（如有预警变更）
4. 更新 `data/companies/` 对应档案（如有重大变化）
5. 更新 `data/index.md`

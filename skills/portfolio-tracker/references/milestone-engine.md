# 里程碑引擎

> 定义里程碑的设置标准、进度追踪和异常检测规则。

## 里程碑分类

| 类型 | 示例 | 权重 | 追踪频率 |
|------|------|------|---------|
| 财务 | 营收达到¥X / 毛利达到X% / 盈亏平衡 | 高 | 月度 |
| 产品 | 产品上线 / 版本发布 / 技术验证 | 中 | 按阶段 |
| 融资 | 下一轮关闭 / 条款书签署 | 高 | 季度 |
| 团队 | 核心岗位到岗 / 团队扩张到N人 | 中 | 按阶段 |
| 客户 | 客户数达到N / 大客户签约 | 高 | 月度 |
| 退出 | 启动IPO / 与买方谈判 | 高 | 按阶段 |

## 里程碑设定原则

1. **SMART原则** — 具体、可量化、可达成、相关、有时限
2. **3-5个核心里程碑** — 不超过5个，避免分散
3. **关联投资条件** — 对赌条款中的里程碑自动置顶
4. **滚动设定** — 当前里程碑完成60%时，引导设定下个阶段

## 进度检测规则

### 正常
- 里程碑在预期时间 ±15% 范围内完成
- 进度按用户提供的实际数据更新

### 延迟判定
| 延迟幅度 | 标记 | 自动操作 |
|---------|------|---------|
| <1个月 | 🟡 轻微延迟 | 记录，不预警 |
| 1-3个月 | 🟠 明显延迟 | 记录 + 追问原因 + 看板标记 |
| >3个月 | 🔴 严重延迟 | 记录 + 预警 + 升级关注 |

### 完成确认
当用户报告里程碑完成时：
1. 确认完成日期和关键数据
2. 记录到公司档案 timeline
3. 自动检查下个里程碑是否到期
4. 如已完成全部里程碑 → 引导设定新阶段里程碑

## 里程碑异常模式

| 模式 | 特征 | 建议行动 |
|------|------|---------|
| 连续跳票 | 2个以上里程碑连续延迟 | 与创始人深度沟通，重新评估可行性 |
| 里程碑模糊化 | 里程碑描述从具体变为模糊 | 追问是否有隐藏问题 |
| 节奏突变 | 里程碑完成节奏突然加快/变慢 | 验证数据真实性 |
| 里程碑被放弃 | 主动删除里程碑但不替换 | 询问原因，评估是否 pivoting |

# 人脉管理

## 触发词

- "帮我记一下[姓名]，[公司][职位]" — 新增联系人
- "[姓名]是谁来着" / "查一下[姓名]" — 查询联系人档案
- "帮我找一下[行业]的[角色]" — 按行业/角色搜索
- "做[领域]的[角色]有谁" — 按条件检索
- "我和[姓名]上次聊了什么" — 互动历史
- "[姓名]认识谁" — 关系网络
- "这周见过的人" / "最近见过谁" — 近期互动汇总

## 数据来源

本 skill 的数据来源：

| 来源 | 调用方式 | 说明 |
|------|---------|---------|
| ① 用户输入 | 直接对话 | 口述联系信息、互动记录 |
| ② `data/` 知识库 | rg 搜索 + 索引表 | 关联已建档公司/项目/会议中出现的联系人 |
| ③ `data/contacts/` 自身库 | 直接读取 | 已有联系人档案检索 |

## 工作流

### Phase 1: 新增联系人 [来源: ①用户输入]

用户主动提供联系人信息时执行。

1. **信息提取：** 从用户输入中提取以下信息，缺失项主动询问：
   - 姓名（必填）
   - 公司 / 职位
   - 角色分类：founder / investor / expert / LP / advisor / FA / legal / other
   - 行业/赛道
   - 如何认识的（场合 / 介绍人 / 日期）
   - 联系方式（按需，仅用户主动提供时记录）

2. **知识库关联 [来源: ②知识库]：**
   - rg 搜索 `data/companies/` → 如该公司已建档，自动链接
   - rg 搜索 `data/meetings/` → 如该联系人出现在既往会议纪要中，提示关联
   - rg 搜索 `data/deals/` → 如关联项目审阅，提示关联
   
3. **重复检测：** 搜索 `data/contacts/` 中是否已有同名联系人
   - 已有 → "该联系人已存在，是否更新信息？"
   - 未重复 → 新建

4. **写入知识库：**
   - 写入 `data/contacts/[姓名].md`
   - 更新 `data/contacts/index.md`
   - 更新 `data/index.md`（联系人索引段）

5. **输出确认：**
   ```
   ✅ 已记录 [姓名] — [公司] [职位]
   
   角色：[founder/investor/专家/...]
   行业：[行业]
   介绍人：[姓名]（如有）
   关联：[公司/项目]（如有）
   ```

### Phase 2: 查询联系人 [来源: ①用户输入 + ③自身库]

用户说"[姓名]是谁来着" / "查一下[姓名]"：

1. **搜索：** rg 搜索 `data/contacts/[姓名].md` 或索引表
2. **未找到：**
   - rg 搜索 `data/` 全库 → 可能在其他文件中被提及
   - 输出提及该人的上下文，询问是否建档
3. **找到：** 输出档案摘要

### Phase 3: 条件检索 [来源: ③自身库]

用户说"帮我找一下[行业]的[角色]" / "做[领域]的[角色]有谁"：

1. 加载 `references/search-rules.md`
2. 知识库搜索流程参考 `docs/SEARCH-PROTOCOL.md`
3. **rg 全文搜索（Phase 1）：** `rg -i "{关键词}" data/contacts/ -l --md`
   - 匹配 → 读取每个匹配文件
4. **索引表搜索（Phase 2）：** 如 rg 未匹配，查 `data/contacts/index.md`
5. **无结果 →** "联系人库中暂无符合[条件]的联系人"
6. **有结果 →** 输出清单：

   ```
   找到 N 位 [角色]（[行业]）：
   
   1. [姓名] — [公司] [职位] | 最近互动：YYYY-MM-DD
   2. [姓名] — [公司] [职位] | 最近互动：YYYY-MM-DD
   ...
   ```

### Phase 4: 互动记录 [来源: ①用户输入 + ③自身库]

用户说"我和[姓名]上次聊了什么"：

1. 读取 `data/contacts/[姓名].md` 中的互动记录表
2. 如无记录 → "该联系人暂无互动记录"
3. 如有 → 输出最近 3-5 条互动：

   ```
   [姓名] 互动时间线：
   
   2026-07-15 — [类型]
   [摘要]
   
   2026-06-20 — [类型]
   [摘要]
   ```

### Phase 5: 关系网络 [来源: ③自身库]

用户说"[姓名]认识谁"：

1. 读取 `data/contacts/[姓名].md` 中的关系网络
2. 对关系网络中的每个名字，读取其档案
3. 输出：

   ```
   [姓名] 的关系网络：
   
   直接关系：
   ├── [姓名2] — [公司]（[关系描述]）
   └── [姓名3] — [公司]（[关系描述]）
   
   交叉关系（共同联系人）：
   ├── [姓名4] — 你和[姓名]都认识
   └── [姓名5] — 你和[姓名]都认识
   ```

### Phase 6: 近期互动汇总

用户说"这周见过的人" / "最近见过谁"：

1. 遍历 `data/contacts/` 中所有联系人的互动记录
2. 筛选最近 N 天的互动
3. 按日期倒序输出

### Phase 7: 自动捕获触发 [← meeting-notes / deal-review 接口]

当 meeting-notes 或 deal-review 完成时，若内容中出现新的人名：

1. 加载 `references/auto-capture.md`
2. 搜索 `data/contacts/` → 是否已记录
3. 未记录 → 自动在 `data/contacts/` 创建基础 stub 档案（仅姓名+公司+出现上下文）
4. 更新 `data/contacts/index.md`

**接口契约：**
- meeting-notes 和 deal-review 不直接写入 contacts/，仅输出人名
- contact-crm 监听输出，若出现未建档联系人则自动创建 stub
- 用户后续可通过 Phase 1 补充完整信息

### Phase 8: 知识库入库

自动更新（用户无需感知）：
1. 新增/更新 `data/contacts/[姓名].md`
2. 更新 `data/contacts/index.md`
3. 更新 `data/index.md`（联系人索引段）

# 模板处理逻辑

## 模式A：保存空白模板

用户上传空白模板文件并说"保存模板"。

### 流程

1. 向用户确认：模板名称、用途、需要填写的字段
2. 查 `data/templates/index.md` → 检查同名模板是否已存在
   - 已存在 → "模板[模板名]已存在，覆盖将丢失原有配置，是否确认覆盖？"
   - 用户确认后继续，否则中止
3. 调用 markitdown CLI 将文件转为 MD（markitdown <输入文件> -o <输出文件>）
4. 分析结构（标题层级、表格、列表、占位符）
5. 原始文件 → `data/templates/original/`
6. 解析文件 → `data/templates/parsed/`
7. 更新 `data/templates/index.md`

## 模式B：从成品文档提取模板 ⭐

用户上传一份已填好的成品文档并说"把这个模板化"。

### 流程

#### Step 1: 格式转换

调用 markitdown CLI 将文件转为 MD（markitdown <输入文件> -o <输出文件>）。

#### Step 2: 结构分析

自动识别文档的结构骨架：

```
固定结构部分（保留为模板框架）：
  - 标题层级（# 会议主题 → 保留为 # [会议主题]）
  - 表格表头（| 序号 | 议题 | 负责人 | → 保留为表头）
  - 列表前缀（- 下一步计划：→ 保留为结构）

可能为变量字段的内容（标记为可填写字段）：
  - 具体日期/时间/地点
  - 姓名/公司名/项目名
  - 金额/数量/百分比
  - 描述性段落（较长文本段）
  - 表格中的数据行
```

#### Step 3: 向用户确认

输出识别结果让用户确认/调整：

```
📋 已分析文档结构，识别到以下可模板化字段：

| # | 原文内容 | 建议字段名 | 类型 | 说明 |
|---|---------|-----------|------|------|
| 1 | 2026-07-16 | 会议日期 | 日期 | — |
| 2 | 张三 | 主持人 | 人名 | — |
| 3 | 半导体赛道分析 | 会议主题 | 文本 | — |
| ... | ... | ... | ... | ... |

✅ 固定结构已保留
⚠️ 标记为变量的内容如上，请确认字段名是否需要调整
```

用户确认后，生成模板解析文件。

#### Step 4: 保存

1. 查 `data/templates/index.md` → 检查同名模板是否已存在
   - 已存在 → 提示覆盖风险，用户确认后继续
2. 原始文件 → `data/templates/original/[模板名].docx`
3. 解析文件 → `data/templates/parsed/[模板名].md`
4. 更新 `data/templates/index.md`

## 解析文件格式

文件头部自动加 YAML header（参考 `docs/DATA-STANDARD.md` §3.6）。

```markdown
# 模板: [模板名]

## 基本信息
- 原始文件: 文件名
- 上传日期: YYYY-MM-DD
- 格式: Word/Excel/PPT
- 用途: [用户描述的用途]

## 结构分析

### 标题区
- [模板的标题结构描述]

### 填写字段
| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| [字段1] | 文本/日期/人名/金额/列表 | [说明] | [示例值] |

### 正文结构
1. 章节1
2. 章节2
...

### 表格结构（如有）
| 原表头 | 列说明 | 是否变量列 |
|--------|--------|-----------|
| [表头1] | [说明] | 是/否 |

## 生成指引
1. 先向用户收集各字段的值
2. 按结构逐项填充
3. 输出格式：按模板结构输出的 Markdown
```

# 会议提取器

> 定义从原始输入中提取结构化会议信息的规则。

## 会议类型识别

| 类型 | 关键词 | 输出侧重 |
|------|--------|---------|
| 路演 | "路演""pitch""BP""融资" | 关键信号 + 评估结论 + 跟进建议 |
| 投委会 | "投委会""IC""决策会" | 决策 + 反对意见 + 条件 |
| 投后管理 | "董事会""季度会""进展同步" | 里程碑 + 财务状况 + 待办 |
| 行业研讨 | "研讨会""专家分享""行业" | 洞察 + 可投资的标的信号 |
| 日常通话 | "call""电话""聊了" | 行动项 + 关系维护 |

## 提取优先级

1. **决策/结论** — 最重要，优先提取
2. **待办事项** — 谁在什么时间前做什么
3. **关键数据** — 提及的数字/比例/时间
4. **情感信号** — 乐观/悲观/犹豫/紧迫
5. **新增信息** — 知识库中未有的新信息

## 公司与赛道识别

提取时自动扫描以下模式：
- 公司名 → 查 `data/companies/` 是否有对应档案
- 赛道/行业名 → 查 `data/sectors/` 是否有分析记录
- 项目名 → 查 `data/deals/index.md` 是否有审阅记录
- 人名 → 查是否已有联系人记录（预留 contact-crm 接口）

## 行动项格式

每条行动项包含：
```
- [负责人]：[具体行动描述] — [截止时间] | [状态：待办/进行中/已完成/逾期]
```

状态自动推断规则：
- 截止时间 > 当前时间 → 待办
- 截止时间 < 当前时间 → 逾期（除非用户说已完成）
- 用户说"做完了/搞定了" → 已完成
- 用户说"还在做/进行中" → 进行中

## 关键信号规则（路演/项目会议）

| 信号 | 标记 | 写入位置 |
|------|------|---------|
| 创始人说了但BP没写的信息 | 🟢 | 写入 deals/ 追加 |
| 团队核心成员有缺失 | 🔴 | 写入 deals/ 追加 |
| 市场判断与赛道分析矛盾 | 🟡 | 写入 sectors/ 追加 |
| 投资人提出的关键追问 | ⚪ | 写入 deals/ 追加 |
| 创始人回答质量 | 📊 | 1-5 分制，写入 deals/ |

# 会议纪要

## 触发词

- "帮我记一下今天的会" / "记录一下会议" — 新建会议纪要
- "刚才和XX公司的会，帮我整理一下" — 会后整理
- "上次XX会议的待办事项怎么样了" — 追踪行动项
- "帮我查一下关于[话题]的会议记录" — 检索历史纪要
- "路演纪要" / "投委会纪要" — 按类型新建

## 数据来源

本 skill 的数据来源：

| 来源 | 调用方式 | 说明 |
|------|---------|---------|
| ① 用户输入 | 直接对话或粘贴文本 | 口述会议内容、粘贴笔记、上传录音 |
| ② `data/` 知识库 | rg 搜索 + 索引表 | 关联已建档的公司/赛道/项目 |
| ③ `data/meetings/` 自身库 | 直接读取 | 历史纪要检索 + 行动项追踪 |

## 工作流

### Phase 1: 信息采集 [来源: ①用户输入]

根据用户输入形式执行对应流程：

**场景 A — 用户口述/粘贴笔记：**
1. 识别会议类型：路演 / 投委会 / 投后管理 / 行业研讨 / 日常通话
2. 询问关键信息（自动识别则跳过）：
   - 会议时间 / 参会方 / 核心议题
3. 接收内容并进入 Phase 2

**场景 B — 用户上传录音/会议文字稿：**
1. 通过 [skill:research-digest] 转换文件为 MD
2. 加载 `references/meeting-extractor.md` 提取结构化信息
3. 进入 Phase 2

### Phase 2: 结构化提取 [来源: ①用户输入]

加载 `references/meeting-extractor.md`，从内容中提取：

```
1. 会议基本信息
   - 类型：[路演/投委会/投后/研讨/通话]
   - 时间：YYYY-MM-DD
   - 公司/项目：（如适用）
   - 参会方：（如提供）

2. 核心讨论
   - 议题1：摘要
   - 议题2：摘要

3. 决策/结论
   - 决定1：
   - 决定2：

4. 待办事项
   - [负责人] [事项] [截止时间]

5. 关键信号（针对路演/项目会议）
   - 🟢 / 🟡 / 🔴 标记

6. 与知识库的关联维度
   - 关联公司：
   - 关联赛道：
   - 关联项目：
```

### Phase 3: 用户确认/补充

1. 输出提取的结构化内容给用户确认
2. 用户可补充遗漏的关键信息
3. 如有关联对象（公司/赛道/项目），系统自动检测并提供追加选项

### Phase 4: 知识库关联 [来源: ②知识库]

自动执行以下关联操作（用户无需感知）：

1. **关联公司：** 如会议提及已知公司 → 更新 `data/companies/[公司名].md`
   - 追加会议纪要链接到该公司档案的"会议记录"部分
2. **关联赛道：** 如会议涉及已知赛道 → 更新 `data/sectors/[赛道名].md`
   - 追加关键信息和来源标注
3. **关联项目：** 如为项目路演 → 更新 `data/deals/[项目名].md`
   - 追加会议纪要反映的评估变化
4. **关联投后：** 如为投后管理会议 → 更新 `data/portfolio/[公司名].md`
   - 追加董事会纪要/里程碑状态更新

### Phase 5: 写入知识库 [来源: ③自身库]

写入 `data/meetings/[YYYYMMDD]_[标题].md`
更新 `data/meetings/index.md`
更新 `data/index.md`（会议索引段）

### Phase 6: 交付

```

━━━ 基本信息 ━━━━━━━━━━━━━━━━━━━━
类型：[路演/投委会/投后/研讨/通话]
公司/项目：[名称]
参会方：[按需]
━━━ 核心讨论 ━━━━━━━━━━━━━━━━━━━━
1. [议题]
   [摘要]

━━━ 决策/结论 ━━━━━━━━━━━━━━━━━━━━
• [决定]

━━━ 待办事项 ━━━━━━━━━━━━━━━━━━━━
• [负责人]：[事项] — [截止]
• [负责人]：[事项] — [截止]

━━━ 关键信号 ━━━━━━━━━━━━━━━━━━━━

```

**输出方式：** 内容直接输出在会话中供用户阅读。

### Phase 7: 行动项追踪

用户说"上次XX会议的待办事项怎么样了"：

1. 搜索 `data/meetings/` 匹配会议
2. 提取待办事项清单
3. 输出：
   ```
   📋 [会议] 待办事项追踪
   
   已完成：
   • [事项] — ✅（已完成日期）
   
   进行中：
   • [事项] — 🔄（截止日期）
   
   逾期：
   • [事项] — 🔴（原定截止日期）
   ```
4. 让用户逐项更新状态
5. 更新会议文件中的待办状态

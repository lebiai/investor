# 体裁模板库

## 公众号文章

- **字数**: 1500-2500 字
- **结构**:
  1. **钩子（前 100 字）**: 一个反直觉的数据/反差/热点切入
  2. **背景（200 字）**: 为什么这个话题现在值得关注
  3. **数据分析（600-1000 字）**: 2-3 个核心数据点，每个拆开讲
  4. **观点输出（300-500 字）**: 投资人的独家判断
  5. **总结 + 互动（100 字）**: 一句话总结 + 抛问题给读者
- **风格**: 专业但不枯燥，数据多但有叙事感

## 小红书

- **字数**: 300-500 字
- **结构**:
  1. **金句开头（1 行）**: 戳中痛点的观点
  2. **3 个核心点**（每个 50-100 字）:
     - 点 1: 数据/事实
     - 点 2: 观点/判断
     - 点 3: 可行动建议
  3. **总结句（1 行）**
  4. **标签（3-5 个）**: #投资人视角 #行业洞察 等
- **风格**: 口语化、有记忆点、适合截图传播

## 朋友圈

- **字数**: 100-200 字
- **结构**: 一句观点 + 一个数据支撑 + 一句号召
- **风格**: 犀利、个人化、像真人在说话

## 投资备忘录

- **字数**: 2000-4000 字
- **结构**:
  1. **投资逻辑（500-800 字）**: 为什么是这个赛道/公司
  2. **关键假设（500-800 字）**: 增长假设/利润率假设/估值假设
  3. **风险分析（300-500 字）**: Top 3 风险 + 触发条件
  4. **估值分析（300-500 字）**: DCF / 可比公司 / 历史区间
  5. **结论 + 仓位建议（200 字）**
- **风格**: 严谨、可回溯、作为决策记录

## 路演纪要

- **字数**: 500-1000 字
- **结构**:
  1. **核心论点（100 字）**: 一句话论点
  2. **3-5 条论据**（每条 50-100 字）
  3. **下一步（50 字）**: 后续跟进事项
- **风格**: 信息密度高、适合快速阅读


## 投委会备忘录（IC Memo）— 审阅驱动版

- **字数**: 3000-5000 字
- **前置依赖**: 先执行 deal-review 审阅，审阅数据自动注入
- **触发方式**: 审阅完成后说"生成为投委会备忘录" / "生成投资备忘录"
- **结构**:

### 封面

```
【机密】投资备忘录 — [项目名]
日期：YYYY-MM-DD
推荐人：[姓名]
推荐评级：⭐⭐⭐ / ⭐⭐ / ⭐ / 暂缓 / 放弃
```

### 1. 执行摘要（300 字）

从 deal-review 的 Layer 1（5 分钟速览）自动提取：
- 一句话定位
- 核心投资逻辑
- 推荐评级 + 关键风险

### 2. 公司全景（500 字）

从 deal-review 的 Phase 2（结构化提取）+ Phase 4（竞品基线）自动注入：
- 产品 / 技术 / 商业模式
- 市场规模与增长驱动
- 竞争格局与定位
- 创始人团队评估（Phase 5 创始人穿透输出）

### 3. 六角色评分雷达（400 字）

从 deal-review Phase 3 自动注入，以文本形式呈现六角色评分：

| 角色 | 评分 | 置信度 | 核心判断 |
|------|------|--------|---------|
| 行业研究员 | X/10 | 高/中/低 | 一句话判断 |
| 财务分析师 | X/10 | 高/中/低 | 一句话判断 |
| 风控官 | X/10 | 高/中/低 | 一句话判断 |
| 产业资本 | X/10 | 高/中/低 | 一句话判断 |
| 竞争对手 | X/10 | 高/中/低 | 一句话判断 |
| 投委会主席 | X/10 | 高/中/低 | 一句话综合判断 |

**辩论焦点：** [从 Phase 3 跨角色批驳中提取 1-2 个核心分歧，附简要分析]

### 4. 关键假设与验证（600 字）

从 deal-review Phase 2+3 自动提取，结构化为：

| 假设 | 依据 | 风险度 | 验证方式 |
|------|------|--------|---------|
| [假设1] | [BP数据/外部验证/行业均值] | 🔴🟡🟢 | [需验证什么] |
| [假设2] | ... | ... | ... |

标注每个假设的数据来源：
- 📄 BP 中声称 → 需独立验证
- 📊 行业公开数据 → 可信
- 🔍 agent-reach 搜索结果 → 标注时效性

### 5. 信号分析（400 字）

从 deal-review 信号库（signal-library.md）的管道数据自动注入：

| 类别 | 数量 | 代表性信号 |
|------|------|-----------|
| 🔴 陷阱 | N | 最强1-2个 |
| 🟢 闪光 | N | 最强1个 |
| 🟡 争议 | N | 最需关注1个 |
| ⚪ 盲区 | N | 缺口 |

### 6. 估值分析（500 字）

从 deal-review Phase 6 自动注入：
- 可比公司估值区间
- 近期交易估值参考
- DCF 敏感度分析（如适用）
- 建议估值区间

### 7. 风险与缓释（400 字）

从 deal-review Phase 3（风控官）+ Phase 7 的尽调路线图提取：

| 风险 | 严重度 | 发生概率 | 缓释措施 | 尽调行动 |
|------|--------|---------|---------|---------|
| [风险1] | 🔴🟡 | 高/中/低 | [当前] | [尽调中需要验证] |

### 8. 尽调路线图（300 字）

从 deal-review Phase 7 自动注入：
- 优先尽调项（Next 30 天）
- 次优先项（30-60 天）
- 创始人追问清单（核心 N 个问题）

### 9. 建议与行动计划（200 字）

综合所有评估后的具体建议：
- 推进 / 观望 / 放弃
- 建议仓位（如适用）
- 下步行动 + 时间表

---

**生成规则：**
- 数据来源标注规则：[来源: deal-review Phase N] 或 [来源: 知识库]
- 如某些 section 因审阅不完整而缺失 → 标注"待补充：需先完成[具体 phase]"
- 所有估值 / 风险信息标注置信度
- 最终输出为 MD 格式，直接呈现在会话中；同步保存到 `outputs/`

# 公司层扫描逻辑

在六维赛道分析 + 六机构评估之后执行，对赛道内的具体公司进行扫描和对比。

## 扫描范围

### 上市公司

- 按子领域/细分环节分类（如半导体：设备/设计/封测/材料/EDA）
- 每家公司收集：
  - 股票代码、市值
  - 营收（近 3 年）、营收增速
  - 毛利率、净利率
  - PE（TTM）/ PS
  - 核心产品/市场地位
- 输出横向对比表

### 未上市公司

- 按细分领域归类
- 每家公司收集：
  - 一句话定位（做什么的）
  - 最新估值 / 融资轮次
  - 主要投资方
  - 成立时间
- 输出项目清单

## 产业链定位

对每一家公司标注：
- 处于产业链哪个环节（上游/中游/下游）
- 可替代性（高/中/低）— 谁卡住关键环节
- 该环节的价值分配占比

## 投资信号生成

结合赛道阶段给出信号类型：

| 赛道阶段 | 建议策略 | 关注方向 |
|---------|---------|---------|
| 早期（渗透率 <5%） | 广撒网 | 找到第一梯队所有玩家 |
| 成长期（5-30%） | 选龙头 | 份额扩张最快的公司 |
| 成熟期（>30%） | 等低估 | 低估值 + 高分红或整合逻辑 |
| 变革期（技术替代） | 关注新势力 | 有颠覆潜力的新入局者 |

## 数据源

```bash
# 搜索上市公司数据

# 搜索未上市公司融资
```

### 上市公司数据

| 数据项 | 优先来源 |
|--------|---------|
| 营收/毛利率/净利率 | 最新年报 + 近3年趋势 |
| 市值/PE/PS | 实时行情数据 |
| 市场地位/产品线 | 年报业务分项 + 行业报告 |
| 研发投入/专利 | 年报 + 公开专利数据 |

### 未上市公司数据

| 数据项 | 来源 |
|--------|------|
| 融资轮次/金额 | 公开融资报道、36Kr、IT桔子、Crunchbase |
| 估值 | 公开披露的投后估值，或可比公司法估算 |
| 投资方 | 融资新闻 + 投资机构官网 |
| 产品/定位 | 公司官网 + 媒体报道 |

### 通用规则

1. 每个数据点标注来源 — "[来源：xx]"
2. 无来源的数据 — 标注"基于公开可比数据估算"
3. 数据矛盾 — 并列出：如"A报道称融资1亿，B报道称3亿"
4. 找不到数据 — 开诚布公说"公开信息不足"，不编造

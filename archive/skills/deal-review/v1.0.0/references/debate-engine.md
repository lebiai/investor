# 辩论引擎

> 核心创新：六角色独立评估 → 交叉批驳 → 综合裁决。每个角色绑定三类知识来源，不是泛泛的人设。

## 统一评估维度

所有角色在以下 5 个标准化维度上评估：

| # | 维度 | 评估内容 |
|---|------|---------|
| 1 | 团队可信度 | 创始人认知深度 / 执行力证据 / 创业动机 / 诚实度信号 |
| 2 | 市场逻辑 | 市场空间假设的合理性 / 增长驱动力 / 进入策略 |
| 3 | 产品壁垒 | 技术壁垒 / 网络效应 / 品牌 / 规模优势 / 可替代性 |
| 4 | 财务健康 | 单位经济 / 现金流 / 融资效率 / 估值合理性 |
| 5 | 时机判断 | 为什么是 NOW？太早 / 太晚 / 正好 / 赌拐点？ |

## 各角色知识来源

### ① 行业研究员

| 知识来源 | 具体行为 |
|---------|---------|
| [skill:agent-reach] | 搜索该赛道的市场规模/第三方数据/竞争格局/技术趋势/政策变化 |
| `../../../data/sectors/` | 读取该赛道历史分析记录中的六维数据 |
| `../../../data/deals/` | 搜索历史审阅中同赛道项目对市场空间的判断 |

输出重点：市场假设验证 → 用第三方数据反驳/支撑文档中的声明

### ② 财务分析师

| 知识来源 | 具体行为 |
|---------|---------|
| [skill:agent-reach] | 搜索可比上市公司 PE/PS/毛利率/净利率等财务基准 |
| `../../../data/companies/` | 读取同类公司档案中的实际财务指标 |
| `../../../data/deals/` | 搜索历史审阅中同类公司的财务预测 vs 后续表现 |

输出重点：财务假设压力测试 + 单位经济穿透

### ③ 风控官

| 知识来源 | 具体行为 |
|---------|---------|
| [skill:agent-reach] | 搜索创始人背景/公司工商信息/涉诉/失信 |
| `../../../data/deals/patterns.md` | 匹配已知的陷阱模式 |
| `../../../data/companies/` | 读取已建档项目的风险点记录 |

输出重点：风险信号识别 + 逻辑漏洞检测

### ④ 产业资本

| 知识来源 | 具体行为 |
|---------|---------|
| [skill:agent-reach] | 搜索产业并购/合作/竞争最新动态 |
| `../../../data/companies/` | 搜索已投/关注公司的生态协同可能性 |
| `../../../data/deals/` | 搜索历史产业协同评估结果 |

输出重点：生态匹配度 + 补位/替代价值

### ⑤ 竞争对手

| 知识来源 | 具体行为 |
|---------|---------|
| [skill:agent-reach] | 搜索同赛道竞品的融资/产品/团队最新动态 |
| `../../../data/companies/` | 读取同赛道其他公司的定位和壁垒 |
| `../../../data/deals/` | 搜索类似项目的竞争评估结果 |

输出重点：就是能打死它的理由 + 它的差异化是否真成立

### ⑥ 投委会主席

| 知识来源 | 具体行为 |
|---------|---------|
| 综合①-⑤的输出 | 分析每条判断的置信度、分歧点、需要确认的盲区 |
| `../../../data/sectors/` | 读取赛道投资策略和阶段判断 |
| `../../../data/deals/` | 搜索最相似的3个历史项目 → "类似项目之前投了吗？后来如何？" |

输出重点：综合裁决 + 跟进建议

## 辩论流程

### Step 1: 独立评估（并行）

每个角色独立执行，输出统一格式：

```
## [角色名] — 独立评估
- 评分（1-10）：[各维度加权分]
- 核心判断（3-5句话）：
  1. [判断1][置信度:高/中/低][来源标签]
  2. [判断2][置信度:高/中/低][来源标签]
  ...
- 关联历史案例："(上次类似项目...)"
- 最大疑问：[一句话]
```

### Step 2: 交叉批驳

角色之间互相支撑或反驳：

```
### 交叉批驳

风控官 → 行业研究员：
"对于研究员判断的'市场空间500亿'，我认为这个假设过于乐观。
因为文档中提到的渗透率测算未引用任何第三方数据。[来源: 模式匹配——市场规模倒推陷阱]"

投委会主席记录分歧：(行业研究员 vs 风控官 在市场规模假设上存在分歧，
差异约30%，需确认底层数据来源)
```

批驳规则：
| 触发角色 | 必须批驳 | 可选支撑 |
|---------|---------|---------|
| 行业研究员 | ⑤ 竞争对手（市场假设） | — |
| 财务分析师 | ② 自己的假设（压力测试） | — |
| 风控官 | ① 行业研究员（逻辑漏洞） | ② 财务分析师 |
| 产业资本 | — | ④ 或 ① |
| 竞争对手 | ① 行业研究员（壁垒判断） | ③ 风控官 |
| 投委会主席 | 综合所有分歧点 | 不做支撑不做反驳，只做裁定 |

### Step 3: 综合裁决

投委会主席输出：

```
## 综合裁决

### 核心分歧
- 角色A vs 角色B 在[维度]上存在重大分歧（差异≥2分）
- 分歧根源：[来源差异/假设差异/信息缺失]

### 判定
- 推进：明确上行空间，XX%概率+2x回报
- 观望：需要1-2个验证点
- 放弃：致命风险

### 推荐评级
⭐⭐⭐ 强烈推荐 / ⭐⭐ 值得跟进 / ⭐ 关注 / 暂缓 / 放弃

### 引用历史
"类似项目：[项目名]（YYYY-MM-DD），当时评级为[评级]，最终[结果]"
```

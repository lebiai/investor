# 投资人工作台

## 触发词

- "开工"
- "投资助手"
- "有什么能力"

## 工作流

用户说出触发词时，输出以下菜单：

```
┌─────────────────────────────────────────────┐
│  📋 投资人助手已就绪                            │
│                                              │
│  ① 赛道分析                                    │
│     → "帮我分析一下[赛道名]"                      │
│     → 六维分析 + 多机构辩论 + 公司扫描             │
│                                              │
│  ② 信息整理                                    │
│     → "帮我整理这篇文章/这个文件"                   │
│     → 自动转MD + 存入知识库                      │
│                                              │
│  ③ 内容产出                                    │
│     → "帮我写一篇关于[话题]的[体裁]"               │
│     → 基于知识库的内容生产                        │
│                                              │
│  ④ 项目扫描                                    │
│     → "帮我扫描[领域]的新项目"                    │
│     → 新融资/新入局者/估值参考                    │
│                                              │
│  ⑤ 知识库浏览                                  │
│     → 终端运行：                                 │
│       cd knowledge-base && npx docsify serve . │
│     浏览器打开 http://localhost:3000            │
│                                              │
│  直接说需求，不用选编号                           │
└─────────────────────────────────────────────┘
```

## 路由规则

根据用户意图路由到对应 skill 的工作流：

- 关键词含"分析""赛道""行业""怎么看" → 激活 investor-sector-analysis
- 关键词含"整理""总结""摘要""读一下" → 激活 investor-research-digest
- 关键词含"写""发""文章""小红书""朋友圈" → 激活 investor-content-prod
- 关键词含"扫描""项目""标的""融资" → 激活 investor-deal-sourcing
- 关键词含"打开""浏览""知识库" → 输出知识库浏览指引
- 无法匹配 → 根据上下文做最佳判断或询问用户
